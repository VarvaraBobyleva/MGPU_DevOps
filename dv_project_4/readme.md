# Отчет по лабораторной работе №4: CI/CD Pipeline для Crypto Dash

## 🎯 Цель работы
Настроить автоматический конвейер (Pipeline) непрерывной интеграции и доставки (CI/CD) для разработанного ранее ETL-скрипта или аналитического приложения. Научиться обеспечивать качество кода (Linting, Testing) и автоматическую сборку Docker-образов при каждом изменении в репозитории.

---

## 1. Настройка CI-инструмента
Для выполнения работы был развернут локальный CI-сервер **Jenkins** в Docker-контейнере. 
* **Конфигурация запуска:** Выполнен проброс Docker-сокета (`/var/run/docker.sock`) для управления хостовым Docker и монтирование тома `jenkins_home`.
* **Доступ:** Интерфейс доступен по адресу `http://localhost:8080`.

> **Результат:** Сервер успешно поднят, плагины установлены, создана задача типа "Pipeline" под названием `CryptoDash_CI`.

---

## 2. Pipeline Script (Jenkinsfile)
В корне проекта `dv_project_4/CryptoDash_CI/` размещен файл `Jenkinsfile` (Declarative Pipeline). Сценарий автоматизирует полный цикл от проверки кода до деплоя.

**Основные стадии пайплайна:**
1. **Checkout**: Клонирование репозитория с GitHub.
2. **Security Check (Quality Gate)**: Аудит безопасности зависимостей Python.
3. **Deploy Infrastructure**: Запуск изолированного контейнера PostgreSQL.
4. **ETL: Load Data**: Автоматическое наполнение базы данных через Python-скрипт.
5. **Build Docker Image**: Сборка Docker-образа дашборда.
6. **Deploy Dashboard**: Запуск финального приложения на порту 8501.

---

Согласно варианту, реализована автоматическая проверка уязвимостей в библиотеках с помощью инструмента `pip-audit`.

В файл `requirements.txt` была намеренно добавлена уязвимая версия библиотеки `requests==2.20.0`. 
* **Результат:** Пайплайн остановил сборку на стадии `Security Check`. В логах обнаружено 17 известных уязвимостей. Сборка помечена статусом **FAILURE**.

После успешного прохождения Quality Gate система автоматически собирает Docker-образ аналитического приложения.

* **Команда сборки:** `docker build -t crypto-dash:latest dv_project_4/CryptoDash_CI/`
* **Проверка:** Образ успешно создается и отображается в локальном списке Docker-образов хостовой машины.
